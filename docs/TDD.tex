\section{Introduction}
This document outlines the test-driven development (TDD) approach for the Alien Invaders game implemented in Godot 4.x. The game is a Space Invaders-style shooter with power-ups, enemy AI, and player controls.

\section{Test Implementation Guidelines}

\subsection{Godot Testing Framework}
Tests should be implemented using Godot's built-in testing framework:

\begin{lstlisting}[language=Python] %GDScript is pyhton like so good enough
extends GutTest

func test_player_movement():
    var player = load("res://player_ship.tscn").instantiate()
    add_child(player)
    
    # Simulate input
    Input.action_press("ui_right")
    await get_tree().process_frame
    
    assert_gt(player.velocity.x, 0, "Player should move right")
    
    Input.action_release("ui_right")
    player.queue_free()
\end{lstlisting}

\section{Test Execution Order}

\begin{enumerate}
    \item Projectile tests (lowest dependency)
    \item Power-up tests
    \item Enemy ship tests
    \item Player ship tests
    \item Manager and spawner tests
    \item Integration tests
\end{enumerate}

\section{Success Criteria}

All tests must pass with 100\% coverage on game logic scripts. The game should:
\begin{itemize}
    \item Allow player to move within screen boundaries
    \item Fire projectiles with configurable cooldown
    \item Apply power-ups correctly with timer expiration
    \item Enemies shoot and drop power-ups with RNG
    \item Game state transitions work (win/loss conditions)
    \item Projectiles collide and deal damage
\end{itemize}
